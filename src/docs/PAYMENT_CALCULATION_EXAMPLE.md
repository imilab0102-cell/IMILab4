### 3. РОЗРАХУНКИ

#### Формули:
```
Для фіксованої оплати:
  calculated_amount = payment_value (грн)

Для процентної оплати:
  calculated_amount = (total_amount × payment_value) / 100
```

#### Розрахунок для наряду:

**Етап 1 - Олег:**
```
Тип: Фіксовано 150 грн
Результат: 150 грн
```

**Етап 2 - Ігор:**
```
Тип: 20% від 7 500 грн
Результат: (7500 × 20) / 100 = 1 500 грн
```

**Етап 3 - Сергій:**
```
Тип: 15% від 7 500 грн
Результат: (7500 × 15) / 100 = 1 125 грн
```

### 7. ПЕРЕВАГИ ЦІЄЇ СТРУКТУРИ

✅ **Гнучкість:** Кожен етап може мати свій тип оплати  
✅ **Точність:** Розрахунки виконуються за формулами, без помилок  
✅ **Прозорість:** Кожен технік бачить свій заробіток  
✅ **Масштабованість:** Можна додати будь-яку кількість етапів  
✅ **Облік:** Легко відслідкувати статус та прибутковість наряду  
✅ **Звітність:** Автоматичні фінансові звіти для аналізу
